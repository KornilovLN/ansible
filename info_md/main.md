## Ansible — автоматизация управления конфигурациями, развертывания приложений и других задач управления  ИТ-инфраструктурой.
<br>Он позволяет управлять множеством серверов из одного места, используя простой язык описания конфигураций на основе YAML.

**_Вот основные шаги для начала работы с Ansible:_**

1. Установка Ansible
<br>Установите Ansible на вашу управляющую машину (обычно это ваш локальный компьютер или сервер,
<br>с которого вы будете управлять другими серверами).

```
sudo apt update && sudo apt install ansible -y
```

2. Настройка инвентаря
<br>Создайте файл инвентаря, который будет содержать список серверов, которыми вы хотите управлять.
<br>По умолчанию Ansible использует файл /etc/ansible/hosts, но вы можете создать свой собственный файл.
<br>Вот пример файла инвентаря: hosts

```
[webservers]
server1.example.com
server2.example.com

[dbservers]
db1.example.com
```

3. Написание плейбуков
<br>Плейбуки — это файлы YAML, которые содержат описание задач, которые вы хотите выполнить на ваших серверах.
<br>Вот пример простого плейбука для установки Apache на веб-серверах:
<br>install_apache.yml

```
- name: Install Apache on web servers
  hosts: webservers
  become: yes
  tasks:
    - name: Ensure Apache is installed
      apt:
        name: apache2
        state: present

    - name: Ensure Apache is running
      service:
        name: apache2
        state: started
```

4. Запуск плейбуков
Запустите плейбук с помощью команды ansible-playbook.
```
ansible-playbook install_apache.yml
```

5. Основные концепции
* Модули: Ansible использует модули для выполнения задач. Например, модуль apt используется для управления пакетами на системах с APT.
* Роли: Роли позволяют организовать плейбуки и связанные файлы в структуру, которая облегчает их повторное использование и поддержку.
* Переменные: Ansible позволяет использовать переменные для параметризации плейбуков.

6. Полезные команды
<br>Проверка доступности серверов:
```
ansible all -m ping
```

<br>Выполнение команды на всех серверах:
```
ansible all -a "uptime"
```
